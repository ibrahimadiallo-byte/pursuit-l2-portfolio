## 1. Slack Notification Onboarding

- **Live demo**: [onboardingslacknotification-b2wqbbwrl-ibrahimapursuits-projects.vercel.app](https://onboardingslacknotification.vercel.app)
- **GitHub repo**: [`Slack-Notification`](https://github.com/ibrahimadiallo-byte/Slack-Notification)
- **Tech stack**: TypeScript, HTML, CSS, Vite
- **What it solves**: The problem that I was looking to solve was just briefly providing new users to Slack the ability to select how they would want to be notified right away.
- **Why it matters**: Slack doesn't give you that ability like certain other apps do and it has a complicated format where you have to go to your settings and click all this nonsense just to activate your notifications.
- **Experience**: This is just a simple step flow that anybody could understand in a minute.

## 3. Netflix Recently Viewed Feature

- **Live demo**: https://netflix-recently-viewed-feature.vercel.app/
- **GitHub repo**: [`Netflix-Recently-Viewed-Feature`](https://github.com/yaasameen-maker/Netflix-Recently-Viewed-Feature)
- **Note**: This repository is owned by my teammate for deployment reasons, but I co-built the feature and contributed to the implementation.
- **Tech stack**: React, Vite, JavaScript, CSS, optional Express + PostgreSQL backend
- **Problem**: Netflix's viewing history is hidden deep in settings, takes 6+ clicks to access, offers poor UX when users forget what they were browsing, and provides no quick, prominent section for recently viewed but not necessarily unfinished content.
- **Goal**: This feature is designed to improve user experience, reduce friction, increase user engagement and session time, and demonstrate UX thinking and problem-solving skills.
- **Outcome**: With this feature, users can easily find recently viewed content, re-discover shows they were interested in, and reach their viewing history with fewer clicks.

## 4. AI Grid Orchestrator

- **GitHub repo**: [`AI-Grid-Infrastructure`](https://github.com/ibrahimadiallo-byte/AI-Grid-Infrastructure)
- **Tech stack**: Python, FastAPI, Streamlit, async HTTP clients
- **Summary**: AI-powered build from L2 focused on orchestrating and visualizing real-time grid data from ERCOT (Texas) and ISO-NE (New England) through a FastAPI backend and Streamlit dashboard.
- **Key features**: Live grid metrics by region, rule-based "Green Mode" and "Reliability Mode" thresholds, and automated recommendations for reducing data center load when the grid is under stress.
- **What the UI shows**: Generation fuel mix, price components, grid capacity, imports and peak demand, system conditions, and reliability/green mode rules for different regions.
- **Screenshots**: High-level dashboard views of Maine (ISO-NE) and Texas (ERCOT) grid conditions, including real-time thresholds, alerts, and automated load-control responses.
